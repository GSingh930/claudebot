Users & Limits
🔒
DM Privacy: Direct messages to the bot are completely private. The dashboard has no access to DM content — conversation history is stored in a separate isolated map that is never exported or logged. Only the bot process can see DMs.
✅ Full Access (Allowlisted)
Set via ALLOWED_USERS Railway variable (comma-separated Discord user IDs).
Max tokens / request 4000
Daily token budget 100,000
🔒 Restricted (Everyone else)
Users not on the allowlist. Set limits via Railway variables below.
Max tokens / request 200
Daily token budget 500
Railway Variables to Set
# Comma-separated Discord user IDs for full access
ALLOWED_USERS=123456789012345678,987654321098765432

# Full access limits
MAX_TOKENS_FULL=4000
DAILY_TOKENS_FULL=100000

# Restricted user limits (everyone else)
MAX_TOKENS_RESTRICTED=200
DAILY_TOKENS_RESTRICTED=500
💡 To find a Discord user ID: enable Developer Mode in Discord Settings → right-click any user → Copy User ID
🕛 Scheduled Cleanup Jobs
🌙
Midnight Reset (daily · UTC)
Wipes all daily token counters. Clears ALL in-memory conversation histories — both guild channels and DMs. No message content survives past midnight.
♻️
Stale Session Eviction (every hour)
Any conversation session (guild or DM) that has been inactive for more than 2 hours is evicted from memory immediately — no need to wait for midnight.

Setup Guide
Getting Started
1
Create a Discord Bot
Go to discord.com/developers/applications → New Application → Bot tab → Reset Token → copy your token.
2
Enable Privileged Intents
In the Bot settings, enable Message Content Intent — required to read message text.
3
Invite Bot to Your Server
OAuth2 → URL Generator → Scopes: bot, applications.commands → Permissions: Send Messages, Read Message History, Attach Files → copy and open the invite URL.
4
Get Anthropic API Key
Sign up at console.anthropic.com, go to API Keys, and create a new key.
5
Deploy to Railway
Push the project to GitHub. In Railway → New Project → Deploy from GitHub repo → select your repo. Then go to Variables and add DISCORD_TOKEN and ANTHROPIC_API_KEY. Railway auto-deploys and the bot starts immediately.
6
Test It
In your Discord server, type !claude hello or mention the bot. Attach an Excel file with a prompt for VBA scripts.
🚂 Railway Deployment
# 1. Push to GitHub
git init && git add . && git commit -m "init"
git remote add origin https://github.com/YOU/YOUR-REPO.git
git push -u origin main

# 2. Install Railway CLI (optional)
npm install -g @railway/cli
railway login && railway init && railway up
✅ Railway auto-detects Node.js via nixpacks. The railway.json sets the start command and health check. No Dockerfile needed.
Railway will provide a public URL for your dashboard (e.g. https://your-app.railway.app). The bot connects to Discord via WebSocket — Railway keeps it alive 24/7.
Running Locally
# Copy env example and fill in your keys
cp .env.example .env

# Install and start
npm install && node dashboard.js
